Making Math Meaningful
Placement Quiz — Full Site Breakdown

Live: lordbyronq.github.io/math-placement-quiz · Code: github.com/lordbyronq/math-placement-quiz
In one paragraph: a parent enters their child's age, hands the device over, and the child solves real, auto-graded math problems. The quiz walks a ladder of eight skill checkpoints taken straight from the program's Levels Chart — up on a passed set, down on a failed one — until it finds the boundary of what the child can do. The first checkpoint they can't pass is the level they should start with: the exact rule of the paper chart, automated. No server, no framework, no build step, no data collection.

1 · The user journey — five screens
ScreenAudienceWhat happens
WelcomeParentExplains the quiz (5–10 min), "levels are not grades," pencil & scratch paper note, and one question: the child's age. Age only picks the starting rung — it never affects the result.
Hand-offParent"Now hand the device to your child." For the youngest entry point the copy changes: the parent works with the child (pre-readers can't quiz alone).
QuizChildOne problem per card: big type, progress dots, "I don't know yet" button. Wrong answers are never announced. No timer, no score, no red X's.

2 · The placement ladder — the chart as data

3 · The adaptive engine — how it decides

Seeding

seed = clamp(age − 6, 0, 6) — a 9-year-old starts at checkpoint 3. Deliberately one rung below age expectation so the first set is a likely pass and the child starts with a win. Seed caps at 6, so algebra (checkpoint 7) is always earned by passing 6.

Pass rule

A set is passed with at most one miss (≈80%+). "I don't know yet" is a silent miss.

The walk

  • Pass → step up and test again. Pass checkpoint 7 → the beyond-algebra outcome.
  • Fail → step down. Fail checkpoint K → Level K.
  • Walking up and hit a failure → that rung is the recommendation. Walking down and finally pass → the rung above is the recommendation.

The walk is monotonic, so it always terminates — typically 2–3 checkpoints, 8–15 problems. The unit tests prove the key property: the recommendation is identical whether the child entered at age 5, 9, or 14. Age changes quiz length, never the result.

Grading

TypeRule
numericCommas/whitespace stripped, compared as numbers — 7,344 = 7344. Decimals and negatives supported. Empty input can't be submitted.
choiceExact match. Used for notation-heavy items: fractions, symbols, factored quadratics.
parent-checkParent taps "Yes, they could" / "Not yet"; only yes is correct.

Skill breakdown

Passed checkpoints' skills → "Your child demonstrated." Within the failed checkpoint, a skill whose problems were all correct still counts as demonstrated (partial credit at skill granularity); the rest → "Level N will teach."

4 · The data model — the one file to edit

Everything a curriculum author might change lives in js/data.js:

{
  id: "4",
  outcome: "L4",              // outcome if this is the first failed rung
  parentAssisted: false,
  skills: [
    { id: "4-mult", text: "Multiplies 1- and 2-digit numbers times 2- and 3-digit numbers" },
  ],
  problems: [
    { id: "4-2", skill: "4-mult", type: "numeric", prompt: "34 × 216 =", answer: "7344" },
    { id: "4-4", skill: "4-frac", type: "choice",
      prompt: "Choose the correct symbol:  2/3 ▢ 3/4",
      choices: ["<", ">", "=", "≠"], answer: "<" },
  ],
}

A validation suite guards edits: 8 checkpoints in order, 4–6 problems each, every skill covered, every answer key sane, every outcome linked to a cornerstonecurriculum.com product page. Break a rule and npm test says exactly what's wrong.

5 · The app layer & persistence

js/app.js is the only file that touches the page. It switches screens (moving keyboard focus to each new heading/problem), builds the numeric input or choice buttons per problem, renders progress dots, disables Next until input is a valid number, hides "I don't know yet" on parent-check items, and routes each finished set through the engine to either the interstitial or the results screen.

Persistence: the full quiz state is saved to sessionStorage after every answer, so an accidental refresh resumes at the same problem with the same dots. Per-tab, expires on close, nothing ever sent anywhere. "Start over" clears it.

6 · Design system
  • Palette: cream page (#f5efe6), paper cards (#fdfaf4), deep maroon headings/actions (#8b1a1a). No green/red correctness colors exist — the quiz never signals right/wrong.
  • Type: EB Garamond headings + Georgia body — bookish, matching the main site. Problem text scales fluidly (clamp(1.5rem → 2.4rem)) for young eyes.
  • Layout: one centered 680px column; verified overflow-free at 320–1440px.
  • Motion: one gentle fade-and-rise between screens (compositor-friendly only), disabled under prefers-reduced-motion.
  • Accessibility: semantic sections, labeled inputs, visible focus, focus moved to each new problem, aria-live prompt, fully keyboard-operable.

8 · Testing

npm test (Node 18+, built-in runner, zero dependencies) — 21/21 passing: 6 data-validation tests over all 42 problems and 9 outcomes, and 15 engine tests covering the age→seed table, grading formats, pass-rule boundaries, up/down/floor/ceiling walks, the seed-independence proof across all outcomes, and skill-breakdown edges.

Manual verification on the built site: responsive 320–1440, keyboard-only completion, reduced motion, comma-formatted answers, refresh-resume, and three full scripted journeys (advanced → Calculus note; selective misses → Level 4 with the right skill split; beginner → Level K).

9 · Hosting & updating
  • Host: GitHub Pages serving the main branch of lordbyronq/math-placement-quiz. Free, HTTPS, nothing to maintain.
  • Update: push to main → live in about a minute.
  • Edit problems: change js/data.js, run npm test, push.
  • Custom domain later: e.g. quiz.cornerstonecurriculum.com — one DNS CNAME + one repo setting.
  • All paths are relative, so the same files also work embedded on another host or in an iframe on the main site.
10 · Deliberate design decisions
DecisionWhy
Child works real problemsPlacement reflects demonstrated ability, not parental recollection.
Seed one rung below age expectationFirst set is a likely pass — the child starts with a win.
No right/wrong feedback mid-quizA placement test shouldn't feel like failing; all misses are silent.
"I don't know yet" buttonA lucky guess can move a child up a whole level; this removes the incentive.
Pass = at most one missOne typo or misread shouldn't sink a set a child genuinely owns.
Vanilla JS, no framework, no buildEditable in a text editor for the next decade; nothing to break.
All content in one tested data fileThe curriculum author can tweak problems safely without touching code.
sessionStorage only, no analyticsZero privacy concerns for families; nothing leaves the browser.

v1 limitations: one fixed problem per slot (variant pools are the natural v2), no email capture/printing/analytics (excluded by design), and the beyond-algebra outcome links to Calculus Made Clear since Geometry/Algebra II is availability-pending.
